Flex-sizing

The Flex-Sizing makes it possible to create flexible layouts that fully adapt to the screen.

If you set up everything correctly you won't have to rely on media queries to support different viewports, layouts, and orientations.

This property determines the resizing behavior of flex children across viewport sizes. They can shrink, grow, or remain static.

Filling up Free Space

One of the most challenging aspects of writing CSS is figuring out how to allocate the free space that remains on the screen after the page has been populated with content.

At some viewport sizes, you'll often find there's too much remaining space and you want to fill it with something.

At other viewport sizes, you might find there's not enough space, and the layout breaks in one way or another.

flex-basis

It specifies the initial length of a flexible item.

It specifies how flex items should behave when there's exactly as much space as needed.

The default value of flex-basis is auto, which means that the size of the flex items is calculated using either the width or height of the element (depending on if itâ€™s a row-based or column-based layout).

when the value of flex-basis is something other than auto, it overrides the value of width (or height in case of vertical layouts).

flex-grow

It specifies how much the item will grow relative to the rest of the flexible items inside the same container.

how flex items should behave when there's a surplus of free space (how they should grow).

defines how any extra space in-between flex items should be allocated on the screen.

flex-grow doesn't divide up the entire flex container, only the space that remains after the browser renders all flex items.
If there's no surplus of space, flex-grow has no effect.
